# 统一验收页面规范

> 日期：2026-05-30
> 对应 TODO：M2-4

## 页面：产品管理

### 功能需求

1. **产品列表**：展示所有产品，每行显示名称、价格、分类；支持删除
2. **创建产品表单**：填写名称、价格、分类、描述，提交后添加到列表
3. **状态处理**：加载中、空数据、错误提示

### 表单字段

| 字段 | 类型 | 校验规则 |
|------|------|---------|
| 名称 | 文本 | 必填，1-100字 |
| 价格 | 数字 | 必填，> 0 |
| 分类 | 枚举(电子/服装/食品/书籍/其他) | 必填，下拉选择 |
| 描述 | 多行文本 | 选填，最多2000字 |

### UI 布局

```
+----------------------------------+
|  [标题] 产品管理                   |
+----------------------------------+
|  创建产品                          |
|  名称: [________]                  |
|  价格: [________]                  |
|  分类: [▼ 电子]                    |
|  描述: [________]                  |
|  [提交]                            |
+----------------------------------+
|  产品列表                          |
|  ---------                        |
|  MacBook Pro | ¥12999 | 电子 [删除]|
|  机械键盘     | ¥399   | 电子 [删除]|
|  ---------                        |
|  共 2 项                          |
+----------------------------------+
```

### 状态

- **加载中**：列表区域显示骨架屏或 loading indicator
- **空数据**：列表区域显示"暂无产品"
- **错误**：表单提交失败显示 SnackBar/Toast；列表加载失败显示"加载失败，请重试"
- **成功**：表单提交成功后清空表单，列表自动刷新

### 验收标准

| # | 标准 |
|---|------|
| 1 | 页面启动后自动加载产品列表 |
| 2 | 空列表时显示"暂无产品" |
| 3 | 填写表单并提交后，产品出现在列表中 |
| 4 | 表单校验：名称为空时提示"请输入名称" |
| 5 | 表单校验：价格为 0 或负数时提示"价格必须大于 0" |
| 6 | 点击删除按钮，产品从列表中移除 |
| 7 | 加载过程中显示 loading 状态 |
