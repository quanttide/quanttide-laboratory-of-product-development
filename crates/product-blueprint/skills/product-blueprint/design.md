# 设计视角 · 页面实现描述

> 核心问题：体验是否到位？

---

## 1. 布局结构

描述页面的 HTML 骨架与网格布局体系。

- 整体布局模式：如「左-中-右三栏」「顶部导航+内容+底部」「全屏棋盘」等
- 响应式断点与布局变化：如 `lg:` 断点下从单列变为三列，或移动端折叠侧栏
- 使用的布局系统：CSS Grid、Flexbox、相对/绝对定位

**输出格式**：一句话概括布局骨架，然后列出各区域的响应式行为。

## 2. 组件树

按视觉区域列出页面中的 UI 组件及层级关系。使用缩进或 Markdown 列表展示父子关系。

- 只列出有业务含义的组件（容器、面板、按钮组、状态指示器等）
- 对每个重要组件描述其职责、状态、变化

**输出格式**：树状列表，每行标注组件名与简要职责。

## 3. 样式方案

描述视觉风格如何落地。

- 主题/设计语言：配色、字体、质感（如「水墨风：琥珀色主色、宣纸纹理、墨迹装饰」）
- 样式实现方式：Tailwind 工具类、内联 CSS、CSS 自定义属性、CSS-in-JS 等
- 关键视觉效果实现：纹理背景、渐变、阴影、滤镜等

**输出格式**：主题总览 → 关键样式技术点（每个一两行）。

## 4. 交互与动效

列出所有用户可操作的交互行为及其实现方式。

- 鼠标/触摸/键盘事件绑定
- 核心交互逻辑（如落子、悔棋、切换棋盘）
- 动画与过渡效果（落子动画、状态切换、Toast 提示）
- 状态反馈（当前玩家指示、统计数据更新）

**输出格式**：交互清单，每个交互说明触发方式、行为逻辑、视觉反馈。

## 5. 可访问性与细节

说明在无障碍、输入法、国际化等方面的处理。

- 语义化 HTML、ARIA 属性
- 键盘快捷键支持
- 多语言/本地化考虑

## 完成检查

输出完成后检查：布局、组件、样式、交互、无障碍五个维度是否全部覆盖。如有遗漏，在末尾用一个句子说明。
