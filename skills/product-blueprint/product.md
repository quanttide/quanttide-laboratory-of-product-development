# 产品视角 · 产品设计描述

> 核心问题：做的是不是用户要的？

---

## 1. 产品定位

用一句话说清楚这是什么产品、为谁解决什么问题。

- 产品名称与定位语
- 核心价值主张（一句话）
- 目标用户画像（谁在用它？在什么场景下使用？）

## 2. 功能清单

列出产品的所有功能，按优先级或模块分组。

每个功能描述格式：

- **功能名称**：一句话说明
- **用户价值**：用户获得什么
- **优先级**：P0（核心不可缺）/ P1（重要）/ P2（锦上添花）

区分"核心闭环"功能与"体验增强"功能。

## 3. 用户流程

描述一个典型用户的完整操作路径。

- 用步骤列表写出主流程：从用户打开页面到完成核心任务
- 标注分支与边界情况（如出错、空状态、首次使用）
- 对每个步骤标注用户意图 → 操作 → 系统响应

**输出格式**：

```
1. 用户意图 → 操作 → 系统响应
2. 用户意图 → 操作 → 系统响应
...
```

## 4. 设计决策与取舍

列出重要的产品决策和背后的理由。

每条决策格式：

- **决策**：我们做了什么选择
- **理由**：为什么选这个方案
- **舍弃了什么**：不做什么，为什么
- **后续考量**：什么条件下需要重新审视这个决策

聚焦有争议或有 trade-off 的决策，不罗列 trivial 的选择。

## 5. 内容策略

- 文案风格与语调（如「古风典雅」「简洁直接」）
- 术语表（关键概念的中英文对照与定义）
- 空状态、错误态、边界的文案处理

## 完成检查

确认以下问题都已回答：用户是谁？核心功能是什么？典型流程是否完整？关键决策是否有取舍理由？如有缺失，在末尾用一句话说明。
